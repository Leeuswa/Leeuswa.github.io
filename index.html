소개

최근에는 Spring AI, LLM/RAG 구조를 학습하며 AI 서비스 개발로 역량을 넓히는 중입니다. 새로운 기술을 배우는 것을 두려워하지 않으며, 코드 리뷰와 문서화로 지식을 나누는 문화를 지향합니다.

펫밀리데이 — 반려동물 통합 케어 플랫폼

4인 팀 프로젝트 · Spring Boot 3.5 모놀리식 · 2026.05 – 2026.06 · 동물병원·예약·인증·알림·AI 도메인 담당

동물병원 예약·진료기록부터 쇼핑몰, 중고거래, 커뮤니티, AI 자가진단까지 제공하는 통합 플랫폼입니다. 저는 동물병원·예약 도메인을 중심으로 인증(JWT/OAuth2), SSE 실시간 알림, 카카오맵 연동, Claude 기반 AI 사료추천 등 서비스 전반의 공통 기능을 담당했습니다.

QueryDSL BooleanExpression으로 null 조건이 자동 무시되는 동적 병원 검색을 만들고, 하버사인 거리 계산으로 가까운 병원 순 정렬을 구현했습니다. 소셜 로그인 성공 시 JWT를 HttpOnly 쿠키로 저장하는 흐름과, 사용자별 다중 SSE 연결을 동시성 안전하게 관리하는 구조도 설계했습니다.

문제

동일 예약 슬롯에 동시 요청이 몰리면 중복 예약이 발생할 수 있는 구조

해결

비관적 락(PESSIMISTIC_WRITE)으로 병원 엔티티를 잠근 뒤 예약을 등록하도록 설계

결과

LMS — 학습 관리 시스템 백엔드 API

팀 프로젝트 (타닥타닥 동아리) · 백엔드 · 2025.05 – 2025.12 · 관리자·교사 파트 담당

학생·교사·관리자 역할을 구분한 온라인 학습 관리 시스템의 REST API입니다. 회원가입·로그인, 강의 관리, 수강 신청, 학습 진행도, 과제 등록·제출·채점, 게시판·댓글을 구현했고, Controller–Service–Repository 계층을 분리해 React 프론트엔드와 연동 가능한 구조를 설계했습니다.

역할마다 보유 속성이 달라 사용자 엔티티를 Admin·Instructor·Student로 분리했고, 학생–강의의 다대다 관계는 수강일·진행 상태를 담는 매핑 엔티티로 풀었습니다.

문제

역할별로 접근 범위와 데이터 속성이 달라 단일 User 테이블로는 책임이 뒤섞이는 구조

해결

역할별 엔티티 분리 + 다대다 관계를 매핑 엔티티로 해소, DTO 계층으로 엔티티 노출 차단
