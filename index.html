02 / approach

일하는 방식

기능을 바로 짜기 전에, 어디서 무너질지를 먼저 찾습니다. 아래는 프로젝트를 진행할 때 실제로 거치는 순서입니다.

01 — Understand

어디서 무너질지 먼저 본다

동일 슬롯 중복 예약처럼 "동시에 요청이 몰리면?"을 요구사항 단계에서 미리 짚습니다.

02 — Design

데이터와 계층을 설계한다

회원·병원·예약·리뷰의 연관관계를 JPA로 모델링하고 Controller-Service-Repository로 관심사를 나눕니다.

03 — Build

정합성을 코드로 보장한다

비관적 락, HttpOnly 쿠키 기반 JWT, 동시성 안전한 컬렉션처럼 안전장치를 기능과 함께 구현합니다.

04 — Verify

경계값과 권한을 검증한다

과거 날짜 예약 차단, 병원 소유권 검증 등 예외 흐름을 확인하고 버그를 정리합니다.

03 / toolkit

도구와 쓰임새

아이콘 나열 대신, 각 기술을 실제로 어떤 문제를 푸는 데 썼는지로 적었습니다.

Language / FrameworkJava · Spring Boot
REST API와 서버 구조 설계. 계층형 아키텍처로 관심사를 분리하고 모놀리식 서비스의 뼈대를 잡는다.
PersistenceSpring Data JPA · QueryDSL
관계형 데이터 모델링과 동적 쿼리. null 조건이 자동 무시되는 검색과 연관관계 매핑을 다룬다.
AuthSpring Security · JWT · OAuth2
Stateless 인증과 역할별 접근제어. 소셜 로그인 성공 시 JWT를 HttpOnly 쿠키로 발급한다.
Concurrency / Realtime비관적 락 · SSE · WebSocket
동일 슬롯 중복 예약을 막는 동시성 제어와 사용자별 다중 연결 실시간 알림.
DatabaseMariaDB · MySQL · H2
운영·테스트 환경에 맞춘 관계형 데이터 저장과 HikariCP 커넥션 풀 운용.
AISpring AI · LLM · RAG
Anthropic Claude 기반 맞춤 추천·자가진단 연동. 최근 임베딩·VectorStore 구조로 확장 중.
04 / profile

이수환에 대해

긴 자기소개 대신, 지금 어디에 집중하고 어떻게 일하는지를 짧게 적었습니다.

눈에 잘 안 보이는 인증·동시성·정합성이 결국 서비스의 신뢰를 만든다고 믿는 백엔드 개발자입니다.

Java와 Spring Boot 기반으로 REST API 설계, JPA, MariaDB를 활용한 웹 서비스를 만들어 왔습니다. Controller-Service-Repository 계층을 나누어 기능을 구현하고, 인증·인가와 데이터 흐름을 설계하는 데 관심이 많습니다.

최근에는 Spring AI, OpenAI API, RAG 구조를 학습하며 AI 서비스 개발로 역량을 넓히고 있습니다. 코드 리뷰와 문서화로 지식을 나누는 문화를 지향합니다.
