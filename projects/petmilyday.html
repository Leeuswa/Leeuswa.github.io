01
Context

반려동물 보호자는 병원 예약, 진료기록, 용품 구매, 정보 공유를 서로 다른 서비스에서 처리합니다. 펫밀리데이는 이 흐름을 하나의 플랫폼으로 합치는 것을 목표로 한 팀 프로젝트입니다.

Spring Boot 기반 모놀리식 웹 애플리케이션으로, 계층형 아키텍처로 관심사를 분리하고 서비스는 인터페이스와 구현체를 나누어 결합도를 낮췄습니다. 저는 그중 동물병원·예약 도메인을 중심으로 인증·알림·AI 연동 등 서비스 전반의 공통 기능을 담당했습니다.

02
Problem

예약 도메인에는 조용히 무너지는 지점들이 있었습니다.

  • 중복 예약 — 같은 병원의 동일 시간 슬롯에 요청이 동시에 몰리면 정합성이 깨진다.
  • 인증 상태 관리 — 일반·소셜 로그인을 함께 지원하면서 토큰을 안전하게 저장하고 권한을 분리해야 한다.
  • 실시간성 — 예약 승인 같은 이벤트를 새로고침 없이 사용자에게 전달해야 한다.
  • 검색 — 키워드·진료과목·응급여부·거리 등 여러 조건이 있으나 없을 수도 있는 검색을 유연하게 처리해야 한다.
03
Approach

기능마다 "언제 깨지는가"를 먼저 정하고, 그에 맞는 안전장치를 함께 설계했습니다. 중복 예약은 비관적 락으로, 인증은 Stateless JWT + HttpOnly 쿠키로, 실시간성은 SSE로, 유연한 검색은 QueryDSL 동적 쿼리로 접근했습니다. AI 추천은 기존 로직을 Anthropic Claude 모델로 교체해 품질을 끌어올렸습니다.

04
Implementation

01동물병원 검색 · QueryDSL 동적 쿼리

  • 키워드·진료과목·응급여부 조건을 BooleanExpression으로 조합해 null 조건은 자동 무시되는 동적 검색 구현.
  • 좌표가 있으면 하버사인(Haversine) 거리 계산으로 가까운 병원 순 정렬, 없으면 기본 정렬로 분기.
  • 카카오 지오코딩으로 주소 → 좌표 변환 후 카카오맵에 위치 표시, 목록 페이징 처리.

02예약 시스템 · 동시성 제어

  • 예약 신청·취소·내역 조회와 병원 관리자 승인 흐름 구현.
  • 동일 슬롯 중복 예약을 막기 위해 @Lock(PESSIMISTIC_WRITE) 비관적 락 적용 — 등록 시 병원 엔티티를 락으로 조회(findByIdForUpdate)해 동시 요청 경합에서 정합성 보장.
  • 과거 날짜·시간 예약 차단, 승인 과정에서 병원 소유권 검증.

03진료 / 건강 기록

  • 진료기록·예방접종 이력 조회 기능.
  • 진료기록 미래 날짜 작성 방지 등 입력 검증.

04인증 / 권한 · JWT + OAuth2

  • JWT 토큰 발급·검증 구현 (HMAC-SHA 서명, role 커스텀 클레임, 30분 만료).
  • 소셜 로그인 성공 시 JWT를 발급해 HttpOnly 쿠키(SameSite=Lax)로 저장하는 OAuth2SuccessHandler 구현.
  • 일반회원 → 병원관리자 전환 시 재로그인 처리.

05실시간 알림 · SSE

  • SseEmitter 기반 실시간 알림 구현.
  • 사용자별 다중 연결을 ConcurrentHashMap + CopyOnWriteArrayList로 관리해 동시성 안전 확보.
  • 연결 종료·타임아웃·에러 및 전송 실패한(dead) emitter 자동 정리.

06AI 연동

  • Anthropic Claude(Spring AI) 기반 맞춤 사료 추천 기능.
  • 기존 AI 로직을 Claude 모델로 교체.

07그 외

  • 마이페이지(내가 작성한 병원 리뷰, 진료기록, 예방접종 이력).
  • 주요 목록 화면 페이징, 관리자 검색, 다수 버그 수정, 메인·병원 페이지 UI 개선.

06
Result

하나의 서비스 안에서 회원·인증부터 병원 예약, 커뮤니티, 쇼핑, 중고거래, AI 기능까지 동작하는 통합 플랫폼을 팀으로 완성했습니다. 제가 맡은 예약 도메인은 비관적 락으로 중복 예약을 막고, SSE로 상태 변화를 실시간 전달하도록 마무리했습니다.
