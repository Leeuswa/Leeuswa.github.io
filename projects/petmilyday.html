개요

동물병원 예약·진료기록 관리부터 반려용품 쇼핑몰, 중고거래, 커뮤니티, AI 자가진단까지 하나의 서비스에서 제공하는 반려동물 통합 케어 플랫폼입니다.

Spring Boot 기반 모놀리식 웹 애플리케이션으로, 계층형 아키텍처로 관심사를 분리하고 서비스는 인터페이스와 구현체를 나누어 결합도를 낮췄습니다. 저는 동물병원 및 예약 도메인을 중심으로 인증·알림·AI 연동 등 서비스 전반의 공통 기능을 담당했습니다.

담당 파트 (Leeuswa)

01동물병원 검색 (QueryDSL 동적 쿼리)

  • 키워드·진료과목·응급여부 조건을 BooleanExpression으로 조합해 null 조건은 자동 무시되는 동적 검색 구현
  • 좌표가 있으면 하버사인(Haversine) 거리 계산으로 가까운 병원 순 정렬, 없으면 기본 정렬로 분기
  • 카카오 지오코딩 API로 주소 → 좌표 변환 후 카카오맵에 병원 위치 표시, 목록 페이징 처리

02예약 시스템 (동시성 제어)

  • 예약 신청·취소·내역 조회 및 병원 관리자 승인 흐름 구현
  • 동일 슬롯 중복 예약을 막기 위해 @Lock(PESSIMISTIC_WRITE) 비관적 락 적용 — 등록 시 병원 엔티티를 락으로 조회(findByIdForUpdate)해 동시 요청 경합에서 정합성 보장
  • 과거 날짜·시간 예약 차단, 승인 과정에서 병원 소유권 검증

03진료 / 건강 기록

  • 진료기록·예방접종 이력 조회 기능
  • 진료기록 미래 날짜 작성 방지 등 입력 검증

04인증 / 권한 (JWT + OAuth2)

  • JWT 토큰 발급·검증 구현 (HMAC-SHA 서명, role 커스텀 클레임, 30분 만료)
  • 소셜 로그인 성공 시 JWT를 발급해 HttpOnly 쿠키(SameSite=Lax)로 저장하는 OAuth2SuccessHandler 구현
  • 일반회원 → 병원관리자 전환 시 재로그인 처리

05실시간 알림 (SSE)

  • SseEmitter 기반 실시간 알림 구현
  • 사용자별 다중 연결을 ConcurrentHashMap + CopyOnWriteArrayList로 관리해 동시성 안전 확보
  • 연결 종료·타임아웃·에러 및 전송 실패한(dead) emitter 자동 정리

06AI 연동 (Claude)

  • Anthropic Claude 모델을 Spring AI로 연동해 반려동물 맞춤 사료 추천 기능 구현
  • 기존 AI 로직을 Claude 모델로 교체해 추천 품질 개선

07그 외

  • 마이페이지(내가 작성한 병원 리뷰, 진료기록, 예방접종 이력)
  • 주요 목록 화면 페이징, 관리자 검색 기능, 다수 버그 수정, 메인·병원 페이지 UI 개선

주요 기능 (서비스 전체)

  • 회원 / 인증 — 일반·소셜 로그인, JWT 인증, 역할별 접근제어(USER / 병원관리자 / ADMIN)
  • 동물병원 · 예약 — 병원 검색·필터·리뷰, 예약 승인 흐름, 슬롯 동시성 제어
  • 커뮤니티 — 자유·모임 게시판, 댓글·대댓글·좋아요, 익명 처리, 신고
  • 쇼핑몰 — 상품·장바구니·주문·결제(KakaoPay)·정기구독·리뷰·QnA·위시리스트
  • 중고거래 — 게시글 CRUD, 1:1 실시간 채팅(WebSocket), 매너 점수, 신고
  • AI · 알림 — Claude 기반 자가진단·사료추천, SSE 실시간 알림
  • 관리자 — 대시보드, 회원·병원·예약·콘텐츠 관리, 검색·페이징
