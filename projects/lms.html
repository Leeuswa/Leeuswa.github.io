P.02 — CASE STUDY · 팀 프로젝트 · 2025.05–2025.12

LMS 학습 관리 시스템

학생·교사·관리자 역할을 구분한 온라인 학습 관리 시스템 백엔드 API. 저는 관리자·교사 파트를 맡아 강의·수강·과제·채점·게시판을 REST API로 설계했습니다.

01
Context

학생·교사·관리자가 각기 다른 권한으로 움직이는 온라인 학습 관리 시스템입니다. 인덕대 타닥타닥 동아리에서 Spring Boot와 React로 진행한 팀 프로젝트로, 프론트엔드와 분리된 REST API 백엔드가 필요했습니다.

02
Problem

역할별로 접근할 수 있는 기능이 달랐고, 강의–수강–과제–채점이 서로 연결된 데이터 흐름을 이루고 있었습니다. 프론트엔드와 독립적으로 개발·연동하려면 계층과 응답 구조를 명확히 나눠야 했습니다.

03
Approach

Controller-Service-Repository 계층을 분리하고 REST API 방식으로 프론트엔드와 연동 가능한 구조를 설계했습니다. Spring Security로 역할을 구분하고, 강의·수강·과제·채점을 엔티티 연관관계로 모델링했습니다.

04
Implementation

담당한 관리자·교사 파트 기능:

  • 회원가입 · 로그인
  • 강의 관리
  • 수강 신청
  • 학습 진행도 관리
  • 과제 등록 · 제출 · 채점
  • 게시판 · 댓글
05
Screens
06
Result

프론트엔드와 연동 가능한 역할 기반 학습 관리 백엔드를 팀으로 완성했습니다. 강의부터 채점, 게시판까지 관리자·교사 흐름을 REST API로 안정적으로 제공하도록 마무리했습니다.
