CASE STUDY — P.02

LMS — 학습 관리 시스템 백엔드 API

팀 프로젝트 (인덕대 타닥타닥 동아리) · 백엔드 · 2025.05 – 2025.12

Java 17Spring Boot 3.3Spring Data JPASpring SecurityH2REST APIGradle

개요

학생·교사·관리자 역할을 구분한 온라인 학습 관리 시스템의 백엔드 REST API입니다. 관리자·교사 파트를 담당했고, Controller–Service–Repository 계층을 분리해 React 프론트엔드와 연동 가능한 구조로 설계했습니다.

담당 기능

01회원 / 인증

  • 역할별(관리자/교사/학생) 회원가입·로그인, 아이디 중복 확인
  • Spring Security PasswordEncoder(BCrypt)로 비밀번호 해싱 저장

02강의 관리

  • 강의 생성·수정·조회, 주차별 강의 콘텐츠 등록 및 파일 업로드

03수강 / 학습 진행도

  • 수강 신청·취소, 내 강의실 조회, 학생별 학습 진행 상태 관리
  • 학생–강의 다대다 관계를 StudentLecture 매핑 엔티티로 해소해 수강일·진행 상태 저장

04과제

  • 과제 등록 → 학생 제출 → 채점 흐름 구현, 과제와 제출을 별도 엔티티로 분리(1:N)

05게시판 · 공통

  • 게시글 CRUD, 댓글 작성·삭제
  • GlobalExceptionHandler로 예외 응답 형식 통일, DTO 계층으로 엔티티 노출 차단

설계하면서 고민한 점

  • 역할별 사용자 분리 — 하나의 User 테이블에 role 컬럼을 두는 대신 Admin·Instructor·Student 엔티티를 분리했습니다. 역할마다 보유 속성(학과, 담당 강의 등)이 달라 테이블을 나누는 편이 명확하다고 판단했습니다.
  • REST 분리 구조 — 뷰 렌더링 없이 JSON API로만 응답하도록 구성해 React 프론트엔드와 역할을 분리하고 병렬 개발이 가능하게 했습니다.
